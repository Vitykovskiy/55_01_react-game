**Архитектура FSD**
Основные концепции FSD
1. Слои (Layers) - вертикальное разделение
   - App - запуск приложения (роутинг, провайдеры)
   - Pages - полноценные страницы
   - Widgets - крупные самостоятельные блоки
   - Features - переиспользуемые бизнес-фичи
   - Entities - бизнес-сущности (user, product)
   - Shared - переиспользуемый код

2. Слайсы (Slices) - горизонтальное разделение по доменам
   - Группируют связанный код внутри слоя
   - Не могут импортировать друг друга на одном слое
     
3. Сегменты (Segments) - техническое разделение
    - ui: компоненты, стили
    - api: запросы к бэкенду
    - model: данные, бизнес-логика
    - lib: вспомогательный код
    - config: конфигурация 

4. Импорты
   // Можно импортировать только с нижних слоев
   pages/ → widgets/ → features/ → entities/ → shared/

5. Публичный API
   - Каждый слайс/сегмент экспортирует только через index.ts

6. Пример:
   src/
   ├── app/                    # Инициализация приложения
   │   ├── routes/            # Маршрутизация
   │   ├── styles/            # Глобальные стили
   │   ├── App.tsx            # Собирает ВСЕ слои вместе
   │   └── index.tsx          # Точка входа 
   │
   ├── pages/                 # Страницы
   │   ├── LoginPage          # Пример страницы
   │       ├── model          # Модель страницы
   │       ├── ui             # Компоненты страницы
   │       ├── api            # API слой страницы
   │       └── index.tsx      # Public API
   │   └── index.tsx          # Реэкспорт
   │
   ├── shared/                # Общие модули
   │   ├── ui/                # Кнопки, инпуты - НЕ знают о бизнесе
   │   ├── lib/               # HTTP-клиент, утилиты
   │   └── config/            # Настройки
